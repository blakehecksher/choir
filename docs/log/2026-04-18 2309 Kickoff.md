## Full notes

### Implementation choices worth recording

**Single render loop, state-dispatched.** `tick()` runs at 60fps and branches on `app.state`. Mouth updates run unconditionally (IDLE targets 0, so they stay closed), band amplitude sampling only runs during PLAYING/ENDING, conductor updates early-return in IDLE. Simpler than separate loops per state and avoids rAF churn on transitions.

**Per-mouth lag via ring-buffer.** Each row keeps a 30-frame `Float32Array` of historical amplitudes; each mouth indexes back by a stable `lagFrames` value (0–9 ≈ 0–150ms). This is the literal implementation of spec §2.4's lag; I considered faking lag with per-mouth time constants alone but the buffer gives real temporal offset that shows up as the row "shimmering" rather than pulsing in unison.

**Dual-sine noise rather than Perlin.** Two sines with stable per-mouth phases and mildly different frequencies, summed. Same practical result as Perlin for this scale (±30%-ish) and no library dependency.

**Exponential easing with asymmetric attack/release.** `rate = 1 - exp(-dt / tau)` with tau = ~80ms attack / ~200ms release per spec §4.2. Also varied ±20% per mouth so the section doesn't open in lockstep.

**Figure-eight conductor path with 180° phase-offset dots.** At t=0 both dots sit at center (segment endpoints). At t=0.25 they're at mirrored extremes. The path crosses itself at center, so when the dots pass each other they visually merge and separate — reads as two hands mirroring, which is what the spec calls for.

**Per-beat ease-in-out via cosine.** `pathEased(t)` splits the cycle into 4 beats and cosine-eases position within each beat. This produces the "slower at extremes, faster through middle of each beat" feel spec §4.2 calls out without needing to hand-tune keyframes per beat.

### Transitions

- Natural end: `audio.onended` → ENDING. 2s settle (mouths fade to 0, dots merge over 1.2s starting from the moment ENDING is entered). Automatic promotion to IDLE at 2s mark.
- Shuffle or upload mid-play: fade audio 300ms → update currentSong → enter ENDING. Same settle animation. User clicks dot in IDLE to start the new song.
- Failed load (CORS/network): `audio.onerror` → `skipToNextSong`. `failureStreak` counter increments; if it hits `songs.length` without a successful `playing` event, we bail out to IDLE instead of looping. Reset on first `playing` event.

### Known gap: CORS-verified URLs

Spec §2.6 calls for verifying CORS on every curated URL. I couldn't test that from this environment, so `songs.json` holds plausible archive.org paths. The app handles bad URLs gracefully (silent skip), but the out-of-box experience depends on at least one URL working. Uploads sidestep this entirely.

### Things I intentionally did not do

- No pause/resume. Spec lists it as optional; skipped.
- No progress bar, track info overlay, playlist UI — explicitly out of scope.
- No mobile-specific tuning.
- No favicon. `assets/` stays empty.
- No accessibility affordances beyond `aria-label`s on buttons and the top-level SVG.
